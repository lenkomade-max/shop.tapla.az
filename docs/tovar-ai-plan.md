# Генерация карточек товаров через Vision + NanoBanana

## Цель

Необходимо реализовать систему автоматической генерации профессиональных карточек товаров для маркетплейса на основе одной исходной фотографии товара.

Важно сделать архитектуру гибкой, чтобы в будущем можно было заменить модели (OpenRouter/KAI, Vision-модель, NanoBanana) без переписывания всей логики.

---

# Общий Pipeline

## Шаг 1. Загрузка товара

В админ-панели пользователь загружает:

* оригинальную фотографию товара (обязательно)
* описание товара (если есть)
* дополнительные характеристики (если есть)

Далее весь процесс происходит автоматически.

---

# Шаг 2. Vision-анализ товара

Исходная фотография сначала отправляется в дешевую Vision-модель.

Ее задача — НЕ написать красивый текст, а максимально подробно извлечь информацию из изображения.

Например:

* что находится на фото
* тип товара
* форма
* цвет
* материал
* текстура
* размеры (если можно определить)
* комплектация
* элементы управления
* кнопки
* разъемы
* дисплей
* индикаторы
* бренд
* логотип
* надписи
* расположение элементов
* особенности конструкции
* возможные функции
* упаковка
* аксессуары
* дополнительные предметы на фото
* пример назначения товара

Нужно получить максимально подробное структурированное описание.

Это описание потом используется далее.

---

# Шаг 3. Подготовка промпта

После этого система объединяет:

* описание поставщика
* дополнительные характеристики
* результат Vision
* внутренние инструкции генерации

После чего формируется большой промпт для NanoBanana.

---

# Шаг 4. Генерация карточек

NanoBanana должна создать не одну картинку, а целый комплект изображений.

Пользователь должен иметь возможность выбрать:

* 3 изображения
* 4 изображения
* 5 изображений

Количество задается параметром.

---

# Логика изображений

## Фото №1

Главная карточка товара.

Это самая важная картинка.

Она должна работать как обложка маркетплейса.

Основные требования:

* товар максимально крупный
* товар занимает большую часть изображения
* чистый современный дизайн
* премиальный вид
* если товар требует демонстрации человеком — использовать человека
* если человек не нужен — не использовать
* сильная композиция
* хороший свет
* высокая читаемость
* ощущение дорогого продукта

Допускается:

* большие акцентные заголовки
* короткие преимущества
* бейджи
* иконки
* современные графические элементы

Все тексты исключительно на азербайджанском языке.

---

## Фото №2

Демонстрация использования товара.

Например:

если фен —

человек сушит волосы.

если массажер —

использует массажер.

если лампа —

работает в интерьере.

Если человек не нужен —

создается максимально красивая lifestyle-сцена.

---

## Фото №3

Особенности товара.

Здесь показываются:

* детали
* функции
* преимущества
* крупные элементы
* макро
* разрезы
* схемы
* дополнительные подписи

---

## Фото №4

Дополнительная информация.

Например:

* комплектация
* характеристики
* размеры
* варианты использования
* особенности конструкции

---

## Фото №5

Финальная продающая карточка.

Например:

* преимущества
* почему выбрать
* сравнение
* сценарии использования
* итоговая презентация

---

# Общие требования ко всем изображениям

Каждая картинка должна выглядеть так, словно ее сделал профессиональный дизайнер маркетплейса.

Нельзя делать:

* шаблонный дизайн
* дешевые градиенты
* случайные иконки
* плохую типографику
* перегруженные макеты
* плохую композицию

Нужно ориентироваться на лучшие карточки Amazon, Wildberries, Ozon, AliExpress и современных DTC-брендов.

---

# Работа с текстом

Все тексты должны быть:

* только на азербайджанском языке
* короткими
* легко читаемыми
* продающими
* без перегрузки

Не нужно писать длинные абзацы.

Лучше использовать:

* короткие преимущества
* заголовки
* буллеты
* цифры
* акценты

---

# Работа с товаром

Важно сохранить настоящий внешний вид товара.

Запрещается:

* менять форму товара
* менять конструкцию
* менять цвет товара
* придумывать новые детали

Можно:

* улучшить качество
* сделать красивее освещение
* сделать товар более премиальным
* убрать шум
* улучшить материалы визуально

Но товар должен оставаться тем же самым.

---

# Если информации недостаточно

Если поставщик дал мало информации —

использовать:

* описание поставщика
* результат Vision
* внешний вид товара

для логического определения назначения товара.

При этом нельзя выдумывать функции, которых у товара нет.

---

# Архитектура

В будущем должны легко заменяться:

* Vision-модель
* OpenRouter
* KAI
* NanoBanana

Поэтому логика должна быть разделена на отдельные независимые этапы.

Нельзя делать жесткую привязку к конкретной модели.

---

# Качество промптов

Промпты для NanoBanana не должны быть статическими.

LLM должна самостоятельно собирать оптимальный промпт под каждый конкретный товар.

То есть система должна учитывать:

* тип товара
* категорию
* стиль
* сценарии использования
* необходимость человека в кадре
* необходимость инфографики
* необходимость lifestyle-фото
* необходимость макро
* необходимость размеров
* необходимость показа комплектации

Для разных товаров должны формироваться разные промпты.

Например, карточки для:

* косметики
* бытовой техники
* электроники
* кухни
* спорта
* детских товаров

не должны строиться по одному шаблону.

---

# Что нужно изучить перед реализацией

Перед написанием промптов и логики генерации необходимо изучить лучшие практики генерации карточек товаров с помощью AI.

Особенно обратить внимание на:

* структуру промптов для image generation;
* способы построения многошаговых пайплайнов (Vision → LLM → Image Model);
* лучшие практики инфографики маркетплейсов;
* современные принципы DTC-дизайна;
* приемы композиции, используемые на Amazon, Shopify, Apple и ведущих e-commerce брендах;
* способы автоматического выбора сценариев изображений в зависимости от категории товара.

Задача — не просто сгенерировать красивые изображения, а построить универсальную систему, которая сможет автоматически создавать качественные продающие карточки практически для любой категории товаров.
